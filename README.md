# Zabbix Templates: 1C Enterprise Monitoring

Набор шаблонов для мониторинга инфраструктуры 1С:Предприятие в Zabbix 7.4+.
Включает мониторинг доступности информационных баз (БД) и учета используемых лицензий на серверах приложений.

## Обзор шаблонов

| Шаблон | Назначение | Метод получения данных | Особенности |
| :--- | :--- | :--- | :--- |
| **1C доступность БД** | Проверка запрета сеансов (`sessions-deny`) | System.run (Windows Agent) | Использует утилиту `rac.exe` для проверки конфигурации ИБ. |
| **1C лицензии** | Подсчет активных пользовательских сеансов | System.run (Windows Agent) + HTTP Agent | Агрегирует данные по всем серверам кластера. Требует настройки API Zabbix для подсчета общей суммы. |

---

## 1. 1C доступность БД (`1C access DB`)

Шаблон проверяет, не запрещены ли сеансы в информационных базах (атрибут `sessions-deny`). Если сеансы запрещены, база считается недоступной для пользователей.

### Принцип работы
1. Основной элемент данных считывает файл-манифест `1C_access.txt` на Windows-хосте.
2. Файл содержит JSON-список команд для проверки каждой базы.
3. LLD создает элементы данных, выполняющие команды через `rac.exe`.
4. Результат команды парсится: если найдено `sessions-deny: on` (или аналогичный маркер "off" в зависимости от логики скрипта), возвращается статус недоступности.

### Установка и настройка

1. **Подготовка файла манифеста:**
   На monitored host (Windows) создайте директорию `C:\Program Files\Zabbix Agent\scripts\`.
   Создайте файл `1C_access.txt` со списком баз в формате JSON.

2. **Формат JSON (`1C_access.txt`):**
   ```json
   [
     {
       "name": "BaseName_01",
       "SERV": "1c-server-01",
       "COM": "cmd /c \"\"C:\\Program Files\\1cv8\\8.3.25.1374\\bin\\rac.exe\" 1c-server-01:1545 infobase info --cluster <CLUSTER_UUID> --infobase <IB_UUID> --infobase-user zabbix_agent --infobase-pwd <PASSWORD> config list | findstr \"sessions-deny\""
     },
     {
       "name": "BaseName_02",
       "SERV": "1c-server-02",
       "COM": "cmd /c \"\"C:\\Program Files\\1cv8\\8.3.27.1688\\bin\\rac.exe\" 1c-server-02:1545 infobase info --cluster <CLUSTER_UUID> --infobase <IB_UUID> --infobase-user zabbix_agent --infobase-pwd <PASSWORD> config list | findstr \"sessions-deny\""
     }
   ]
   ```
   *Примечание:* Команда должна возвращать строку, содержащую статус сеансов. Шаблон использует Regex для определения статуса.

3. **Настройка прав доступа:**
   Убедитесь, что пользователь, от имени которого запущен Zabbix Agent (обычно `LOCAL SYSTEM` или специальный пользователь), имеет права на выполнение `rac.exe` и чтение файла `1C_access.txt`.

## 2. 1C лицензии (`1C Licenses`)

Шаблон мониторит количество активных пользовательских сеансов (лицензий) на серверах 1С.

### Принцип работы
1. Основной элемент данных считывает файл-манифест `1C_lic.txt`.
2. LLD создает элементы данных, выполняющие команды подсчета сеансов через `rac.exe session list`.
3. Отдельный элемент данных `1С лицензии всего` обращается к Zabbix API через HTTP Agent, чтобы суммировать значения всех элементов данных с именем "1C Лицензии".

### Установка и настройка

1. **Подготовка файла манифеста:**
   На monitored host (Windows) создайте файл `C:\Program Files\Zabbix Agent\scripts\1C_lic.txt`.

2. **Формат JSON (`1C_lic.txt`):**
   ```json
   [
     {
       "name": "Serv1C_1c-server-01",
       "SERV": "1c-server-01",
       "COM": "cmd /c \"\"C:\\Program Files\\1cv8\\8.3.25.1374\\bin\\rac.exe\" 1c-server-01:1545 session list --cluster <CLUSTER_UUID> | find /c \"1CV8C\""
     },
     {
       "name": "Serv1C_1c-server-02",
       "SERV": "1c-server-02",
       "COM": "cmd /c \"\"C:\\Program Files\\1cv8\\8.3.27.1688\\bin\\rac.exe\" 1c-server-02:1545 session list --cluster <CLUSTER_UUID> --cluster-user admin --cluster-pwd <PASSWORD> | find /c \"1CV8C\""
     }
   ]
   ```
   *Примечание:* Команда использует `find /c "1CV8C"` для подсчета строк с активными клиентскими сеансами.

3. **Настройка HTTP Agent (Суммарное количество):**
   Элемент данных `1С лицензии всего` требует настройки доступа к Zabbix API.
   *   **URL:** Адрес вашего Zabbix Frontend/API (например, `https://zabbix.example.com/api_jsonrpc.php`).
   *   **Authorization:** Замените токен в заголовке `Authorization: Bearer <TOKEN>` на действительный API-токен пользователя с правами чтения элементов данных.
   *   **Params:** В теле запроса (`posts`) проверьте параметр `"group":"1C"`. Убедитесь, что хосты с 1С находятся в группе с таким именем, или измените фильтр в запросе.

## Общие требования

*   **OS:** Windows Server (для агентов 1С).
*   **Zabbix Agent:** Версии 5.0+ (рекомендуется 6.x/7.x).
*   **1C Platform:** Установленная платформа 1С:Предприятие (для наличия `rac.exe`).
*   **Права:** Пользователь Zabbix Agent должен иметь права на запуск `rac.exe` и подключение к портам кластера 1С (по умолчанию 1540-1545, 1560-1591).

## Troubleshooting

### Ошибка выполнения команд (System.run)
1. Проверьте логи Zabbix Agent на наличие ошибок `command is not allowed`.
2. Убедитесь, что путь к `rac.exe` в JSON-файле соответствует установленной версии платформы.
3. Проверьте таймауты: операции с `rac.exe` могут занимать время, особенно при большом количестве сеансов. Увеличьте `Timeout` в конфиге агента и в настройках элемента данных в Zabbix.

### Неверный подсчет лицензий
1. Проверьте команду `find /c "1CV8C"`. Она может отличаться в зависимости от локали или версии 1С.
2. Выполните команду вручную из-под пользователя Zabbix Agent в CMD, чтобы убедиться, что она возвращает число.

### Ошибка HTTP Agent (API Zabbix)
1. Проверьте валидность API токена.
2. Убедитесь, что Zabbix Server имеет сетевой доступ к самому себе (или к URL API) по HTTPS.
3. Проверьте, что группа хостов "1C" существует и содержит нужные хосты.
